# Knowledge Vault — Karpathy LLM Wiki

> 基于 [Andrej Karpathy 的 LLM Wiki 模式](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f) 搭建的个人知识图谱。
>
> 核心思想：**知识是复利增长的**。每次摄入新资料，LLM 将其编译进 Wiki，而不是每次查询都从头推导。

## 三层架构

```
┌─────────────────────────────────────┐
│  Raw Sources (只读, 人工维护)        │
│  articles/ papers/ repos/ ...       │
├─────────────────────────────────────┤
│  The Wiki (LLM 维护, 人类阅读)       │
│  sources/ concepts/ entities/ ...    │
├─────────────────────────────────────┤
│  The Schema (CLAUDE.md)             │
│  告诉 LLM 如何运作的规则文件          │
└─────────────────────────────────────┘
```

## 四大操作

| 操作 | 触发词 | 说明 |
|------|--------|------|
| **Ingest** | "保存到知识库"、"ingest" | 摄入新资料，LLM 提取关键信息并更新 Wiki |
| **Query** | "知识库里有什么"、"查一下 X" | 基于 Wiki 回答问题，附带来源引用 |
| **Synthesize** | "写一篇关于 X 的综述" | 跨来源综合，生成综述报告 |
| **Lint** | "检查知识库健康" | 发现矛盾、孤立页面、过时内容 |

## 快速开始

1. 把资料放入 `raw/` 对应子目录
2. 告诉我："请把 X 摄入知识库"
3. 浏览 `wiki/` 查看 LLM 生成的页面
4. 用 Obsidian 打开本目录，享受图谱视图
