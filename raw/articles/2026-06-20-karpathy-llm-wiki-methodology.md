# Karpathy LLM Wiki — 完整方法论

> 来源: [Andrej Karpathy 的 Gist](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)
> 这是 Karpathy 提出的 LLM Wiki 模式的原始思想文档。

## 核心思想

大多数人的 LLM 文档体验是 RAG：上传文件，LLM 在查询时检索相关片段并生成答案。这可行，但 LLM 每次都在从头重新发现知识，没有积累。

**LLM Wiki 的不同之处**：LLM **增量构建并维护一个持久的 Wiki** — 一个结构化的、相互链接的 Markdown 文件集合，位于你和原始资料之间。当你添加新资料时，LLM 不只是索引它以备后续检索，而是读取它、提取关键信息、整合到现有 Wiki 中。

**关键差异：Wiki 是一个持久的、复利增长的产物。** 交叉引用已经存在，矛盾已经被标记，综合已经反映了你读过的一切。每次添加资料和每次提问，Wiki 都变得更丰富。

## 三层架构

1. **Raw Sources（原始资料）** — 你策源的文档集合。不可变，LLM 只读不写。这是你的真相源。
2. **The Wiki** — LLM 生成的 Markdown 文件目录。摘要、实体页、概念页、对比分析、综述。LLM 完全拥有这一层。
3. **The Schema（模式）** — 告诉 LLM Wiki 如何结构化、有什么约定、工作流是什么的配置文件。这是让 LLM 成为有纪律的 Wiki 维护者的关键。

## 四大操作

### Ingest（摄入）
你把新资料放入 raw 集合，告诉 LLM 处理它。流程：LLM 读取资料 → 与你讨论关键要点 → 在 Wiki 中写摘要页 → 更新索引 → 更新相关实体和概念页 → 追加日志。单个资料可能涉及 10-15 个 Wiki 页面。

### Query（查询）
你针对 Wiki 提问。LLM 搜索相关页面、阅读它们、综合回答并附带引用。**好的回答可以归档回 Wiki 作为新页面** — 这样你的探索也和摄入的资料一样复利增长。

### Lint（健康检查）
定期让 LLM 检查 Wiki 健康：页面间矛盾、过时主张、孤立页面、缺失概念、缺失交叉引用、数据空白。

### Synthesize（综合）
跨多个来源形成综合报告，识别多源共识 vs 单一来源观点，标注知识空白。

## 索引与日志

- **index.md** — 面向内容。Wiki 中所有页面的目录，按类别组织。LLM 在每次摄入时更新。
- **log.md** — 面向时间线。仅追加的操作记录。每条记录以 `## [YYYY-MM-DD]` 开头，可用 grep 解析。

## 为什么这有效

维护知识库最繁琐的部分不是阅读或思考，而是记账。更新交叉引用、保持摘要最新、记录新数据何时与旧主张矛盾、在几十个页面间保持一致性。人类放弃 Wiki 是因为维护负担增长快于价值。LLM 不会厌倦，不会忘记更新交叉引用，一次可以处理 15 个文件。

**人类的工作**是策源、引导分析、提出好问题、思考这一切意味着什么。**LLM 的工作**是其他一切。

## 推荐工具

- **Obsidian Web Clipper** — 浏览器扩展，将网页文章转为 Markdown
- **Obsidian 图谱视图** — 查看 Wiki 的形状
- **Marp** — Markdown 幻灯片格式
- **Dataview** — Obsidian 插件，对 YAML frontmatter 做查询
- **qmd** — Markdown 文件的本地搜索引擎（BM25/向量混合搜索）
- **Git** — 版本历史、分支、协作

## 应用场景

- 个人知识管理：跟踪目标、健康、心理学
- 研究：深入一个主题数周/数月
- 读书：逐章归档，构建角色/主题/情节的 Wiki
- 团队：内部 Wiki，由 Slack 线程、会议转录、项目文档喂养
